```java
class Solution {
    public int search(int[] nums, int target) {
        BinarySearch bs = new BinarySearch();
        return bs.binarySearch(nums,target,0,nums.length);
    }

}

class BinarySearch {
    static public int binarySearch(int[] nums,int target, int low,int high) {
        int mid = (low + high) / 2;
        if (low >= high) {
            return -1;
        }

        if (nums[mid] == target) {
            return mid;
        }
        else if (target < nums[mid]) {
            return binarySearch(nums,target,low,mid);
        }
        else {
            return binarySearch(nums,target,mid+1,high);
        }
    }
}
```

Time Complexity - O(logn)    -> NOT O(nlogn) (that is because we are considering array is already sorted)
Space Complexity - O(logn)

Why is space O(log n)?

Every recursive call stays on the call stack until it returns.

For example:
binarySearch(0,1024)
binarySearch(0,512)
binarySearch(256,512)
binarySearch(384,512)

There are roughly log₂(n) active function calls.

# Recursive Binary Search

You already know this.

```
Search(0, 9)

        mid

Search(0, 4)

     mid

Search(3, 4)

   mid

Found
```

Who remembers the remaining search range?

👉 **The function calls (recursion).**

Every recursive call says:

> "I'll remember this range, you go search inside."

---

# Iterative Binary Search

Now imagine...

There is **only one function**.

No recursive calls.

So who remembers the search range?

👉 **You do.**

Using two variables:

```text
low
high
```

Initially

```text
0              n-1
L---------------H
```

---

## Step 1

Find the middle.

```text
L------M------H
```

Three possibilities:

### Case 1

```
target == nums[mid]
```

Done.

Return.

---

### Case 2

```
target < nums[mid]
```

Everything after mid is useless.

So instead of calling

```
binarySearch(low, mid-1)
```

you simply update

```
high = mid - 1
```

Notice what happened.

Instead of creating a new function,

you simply changed the variables.

---

### Case 3

```
target > nums[mid]
```

Everything before mid is useless.

Update

```
low = mid + 1
```

Again...

No recursive call.

Just changing variables.

---

Then...

Repeat.

How?

With a loop.

---

# The important realization

Recursive version:

```text
search(low, high)

↓

search(low, mid-1)

↓

search(mid+1, high)
```

Iterative version:

```text
low = ...
high = ...

↓

update low/high

↓

repeat
```

It is literally the same algorithm.

The only difference is

```
Function Call
```

becomes

```
Loop
```

---

# Loop Condition

Now think.

When should we stop?

Initially

```
low <= high
```

There is still something to search.

Example

```
0 1 2 3 4

L       H
```

Eventually

```
      H
        L
```

Now

```
low > high
```

Search space is empty.

Target doesn't exist.

Return -1.

---

implemented this - 

```java
class Solution {
    public int search(int[] nums, int target) {
        int low = 0;
        int high = nums.length;
        while (low < high) {
            int mid = (low + high) / 2;
            if (nums[mid] == target) {
                return mid;
            } else if (target < nums[mid]) {
                high = mid;
            } else {
                low = mid+1;
            }
        }
        return -1;
    }

}
```

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        int low = 0;
        int high = nums.length;
        while (low < high) {
            int mid = (low + high) / 2;
            if (nums[mid] == target) {
                return mid;
            }
            else if (target < nums[mid]) {
                if (mid == 0) {
                    return mid;
                }
                if (target > nums[mid-1]) {
                    return mid;
                }
                high = mid;
            }
            else if (target > nums[mid]) {
                if (mid == nums.length-1) {
                    return mid+1;
                }
                if (target < nums[mid+1]) {
                    return mid+1;
                }
                low = mid;
            }
        }
        return low;
    }
}
```

### Biggest realization

The beautiful thing about this problem is:

> **It is literally Binary Search.**

If the target exists:

```
return mid;
```

If it doesn't:

Binary Search eventually shrinks the search window until

```
low == high
```

Guess what `low` is?

👉 **Exactly the insertion position.**

So you don't need to manually check:

* previous element
* next element
* first element
* last element

The algorithm gives it to you automatically.
